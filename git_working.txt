Работа в GIT

1.Настройка консоли
Заходим в файл редактирования BASH через конманду vim .bashrc
Вставляем в конце файла :

# Show a current active git branch in the shell prompt
export PS1='\t \[\033[01;32m\]\u\[\033[01;34m\] \w\[\033[01;33m\]$(__git_ps1)\[\033[01;34m\] \$\[\033[00m\] '

# Shortcat for the pretty git log. Can be extended with the commit count parameter (last -10, last -35)
alias last='git log --graph --all --oneline --decorate '

2. устанавливаем git командой sudo apt-get install git
3. создадим папку где будут храниться проекты GIT. Выполним команду sudo makedir Dev/testproj/ перейдём в папку cd Dev/testproj/
4. проверим что репозитория нет. командой ll. далее инициализируем создание репозитория командой git init
5. создадим файл для версионирования (или скопируем). Выполним комманду touch index.md проверим статус созданного файла выполнив комманду git status
6. добавим все файлы каталога для отселживания выполнив комманду git add .
7. запишем изменения в репозиторий выполнив комманду git commit -m "любой комментари"
8. посмотрим историю выполнив комманду last
9. подцепляем репозиторий к account на github для этого:
	9.1 регистрируемся и создаём учетную запись на www.github.com
	9.2 генерируем ssh ключ, для этого выполним комманду: ssh-keygen и везде со всем согласимся ничего более вводить не нужно везде жмём enter
	9.3 копируем сгенерированный ssh key для этого выполним комманду cat ~/.ssh/id_rsa.pub Примечание команда cat выводит содержимое текстового файла на экран!
	9.4 заходим на github  в настройки (settings) раздел SSH and GPG keys -> NEW SSH KEY вставляем скопированный ключь SSH 
	9.5 заходим на главную страницу github и создаём новый репозиторий (назовём его my_tutor) и исполняем комманды выведеные при создании репозитория на странице github
	9.6 вводим git push origin  HEAD - копируем данные из локали в облако
	
Файлы должны быть перенесены в рабочую область !

Для клонирования чужого git репозитория необходимо выполнить команду: git clone (cсылка репозитория) + название папки в которую хотим поместить скопированные файлы.
Перейдём в каталог и посмотрим историю изменений введя команду last
После создадим еще какой нибуь файл с помощъю команды touch и добавим в отслеживание с помощью команды git add . 
Зафиксируем изменения с помощью команды git commit -m
Перенесём изменения в репозиторий github  с помощью команды git push origin HEAD 

Для того что бы получить актуальные изменения с репозитория возможно воспользоваться следующими командами:
	git pull - обновление локального репозитория совмещает в себе две команды git fetch (получить изменения) + git merge (объединить ветки)
	если после выполнения этих команд проверить логи (команда last) то мы увидим что две ветки объединились.

СОЗДАНИЕ ВЕТОК
для создание новой ветки воспользуемся командой git branch
для примера создадим новую ветку git branch add_order
для того что бы перейти на новую ветку необходимо воспользоваться командой git checkout add_order (в конце указывается ветка на которую  мы хотим перейти)
